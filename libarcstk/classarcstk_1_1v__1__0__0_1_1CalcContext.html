arcstk::v_1_0_0::CalcContext class
#include <calculate.hpp>

Interface for information about the current audio input.

Contents

  • Reference
    • Constructors, destructors, conversion operators
    • Public functions
    • Private functions

Computationally relevant data are TOC information as there is track count, offsets, and leadout. Additionally, the filename is provided.

Determines whether the calculation is single- or multitrack and whether it has to skip samples on the beginning of the first or the end of the last track.

CalcContext also provides service methods for identifying the first and last sample of a track relevant for computation or to get the track for a given sample.

uint32_t arcstk::v_1_0_0::CalcContext::last_relevant_sample(const TrackNo track) const

Service method: Get 0-based index of the last relevant sample of the specified 1-based track.

Parameters
track in 1-based track number, accepts 0 as offset of track 1
Returns Index of last sample contributing to the specified track's ARCS

Note that parameter track is 1-based, which means that last_relevant_sample(2) returns the last 0-based sample of track 2 (and not track 3).

If no offsets are set, the output will always be identical to last_relevant_sample().

For track == 0, the last sample of the offset before track 1 is returned. This may of course be 0 iff the offset of track 1 is 0.

lba_count arcstk::v_1_0_0::CalcContext::length(const uint8_t track) const

Return the normalized length of the specified 0-based track.

Parameters
track in The 0-based track to get the length for
Returns The length for the specified 0-based track

Note that length(i) == offset(i+2) - offset(i+1) for all i: 0 <= i < toc().track_count() - 1.

For the last track t = toc().track_count() - 1, the value of length(t) may be 0 if the TOC is incomplete and the instance is not yet updated with the concrete AudioSize, otherwise it is length(t) == leadout_frame() - offset(t+1).

TrackNo arcstk::v_1_0_0::CalcContext::track(const uint32_t smpl) const

Returns 1-based track number of the track containing the specified 0-based sample.

Parameters
smpl in The sample to get the track for
Returns Track number of the track containing sample smpl

If sample_count() is 0, the method will return 0 regardless of the actual value of smpl.

If smpl is bigger than sample_count() - 1, the method will return an invalid high track number ("infinite").

The user has therefore to check for the validity of the result by checking whether 0 < result <= CDDA.MAX_TRACKCOUNT.

uint8_t arcstk::v_1_0_0::CalcContext::track_count() const

Convenience method: Total number of tracks.

Returns The number of tracks represented in this file

This number will aways be a non-negative integer up to 99 .

Since the track count should be accessed in a uniform way, regardless whether we actually have multiple tracks or only one.

Note that this is independent from is_multi_track(). A TOC containing only one track would have a CalcContext in which track_count() is 1 but is_multi_track() is TRUE. Method is_multi_track() specifies the processing mode while track_count() just provides information about the TOC.
