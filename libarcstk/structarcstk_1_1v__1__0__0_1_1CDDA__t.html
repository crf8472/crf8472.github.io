Variable documentation

const int arcstk::v_1_0_0::CDDA_t::BYTES_PER_FRAME

Number of 2352 bytes per frame.

This follows from CDDA where 1 frame == 588 samples * 4 bytes/sample

const int arcstk::v_1_0_0::CDDA_t::BYTES_PER_SAMPLE

Number of 4 bytes per sample.

This follows from CDDA where 1 sample == 16 bit/sample * 2 channels / 8 bits/byte

const lba_count arcstk::v_1_0_0::CDDA_t::MAX_OFFSET

Maximal valid offset value in cdda frames.

Redbook defines 79:59.74 (MSF) (+leadin+leadout) as maximal play duration which is equivalent to 360.000 frames, thus the maximal offset is frame index 359.999.

const lba_count arcstk::v_1_0_0::CDDA_t::MIN_TRACK_LEN_FRAMES

Minimal number of frames a track contains.

The CDDA conforming minmal track length is 4 seconds including 2 seconds pause but the pause does not contribute to the track lengths, thus 2 sec * 75 frames/sec == 150 frames.

const lba_count arcstk::v_1_0_0::CDDA_t::MIN_TRACK_OFFSET_DIST

Two subsequenct offsets must have a distance of at least this number of frames.

The CDDA conforming minimal track length is 4 seconcs including 2 seconds pause, thus 4 sec * 75 frames/sec == 300 frames.

const int arcstk::v_1_0_0::CDDA_t::SAMPLES_PER_FRAME

Number of 588 samples per frame.

This follows from CDDA where 1 frame == 44100 samples/sec / 75 frames/sec
